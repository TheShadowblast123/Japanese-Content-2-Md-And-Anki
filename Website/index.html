Why Bother making such an app?

Dogma

Before I can explain why I believe this tool is valuable, I must explain my beliefs about the language learning process. I believe that due to the low inherit reward of the process of learning a language, we must find a highly motivating source. Whether it is love, cultural appreciation, a favorite book, or the allure of the end goal, whatever motivates us must be enough to endure a long and arduous process that delays gratification.

I also believe that we learn languages best from the language itself out in its natural habitat. Music, movies, books, comics, animations, social media, articles etc. all the different sources that contain the very language we wish to speak. For natives of the Latin alphabet, reading another language with the latin alphabet is rather easy, often a beginner can get most of the sounds correct and remembering words isn't that much of an issue.

With Kanji, hiragana, and katakana, the aspiring Japanese learner has a lot to hold in working memory for each sentence. Also, Kanji having many meanings and pronunciations means that evenatually this learner will have to expand upon their previous notes with new information. Beyond the belief of motivation and the belief that regular language is the best source to learn from, I'd rather not have much more dogma about language learning.

The Objectives

I wanted something that would allow the learner to be motivated by their biggest motivating factor. Learning through content, allows for this without issue. Whether your desire is the content itself or beyond the content, the content won't interfere with your motivation and often the content at some point becomes yet another source of motivation.

I wanted something that facillitates and encourages engagement with the language. In the end, whether you learn from grammar books or from content, you'll end up studying from content. I thusly want to build something that makes engagement with the language much less intimdating.

I wanted something that lowers the time it takes to gather data that one may need on any given kanji, word, or sentence. Lowering this time is the focus of many tools out there. The more time you spend collecting the data you need to study, the less time you have for actually studying.

I wanted something allows for expansion with the knowledge of the language learner. I didn't want something that would be quickly outgrown and useless within mere moments.

Achieving Goals

This Python script automates the creation of Japanese language learning notes, focusing on sentences, words, and kanji. It utilizes various APIs and markdown files for organizing and managing the notes. Jisho's API (Thank you to the Jisho team) is used to get data and words and kanji. The MeCab Parser isn't 100% accurate, but it's more than enough to get the job done and it means the learner doesn't have to be proficient in Japanese to start studying. Despite its inaccuracies, Google Translate is then used to translate the sentences as it did not require an API key, making it the best option for free and open source usage.

What does it make anyways?

Notes for Words, Kanji and Sentences

Let's take a look at these generated notes. The formatting is such that if Obsidian to Anki ever works with programatically generated cards, it'll work with the markdown generated. But just because there's a generation process, doesn't necesairly mean that one cannot edit the notes. Keeping Everything above the Japanese, "END", "Tags: ", and "Back: ", the user can edit anything else they want including adding images.They get links to any sub notes (For Sentences, words and for words, kanji), a link back to sources, definition/translation/keyword, and pronounciation/readings.

The name of the file is the sentence, word, or kanji itself. This means you can open the kanji folder, and search for a kanji given that you know how to type it, but of course there has to be a better way to quickly find the desired content that you want.

Kanji.md, Sentences.md, Words.md, and Content.md

The name of the file is the sentence, word, or kanji itself. This means you can open the kanji folder, and search for a kanji given that you know how to type it, but of course there has to be a better way to quickly find the desired content that you want. If you know where a desired kanji comes from, for example, find the content file, navigate to the sentence link containing the kanji, click the sentence, click the word, and click the kanji. If you know how to type it, there's dedicated markdown files which are lists of all the sentences, words, and kanji.

How does it work?

Project Structure

Before the addition of csv files for Anki, the project was structured on markdown files and a few folders. Key files were created to act as the databases, so really all the program does is it reads the new inputs and given that it either adds to files in the database or it creates new files. The result is that a user can enter in all the text files that they want to have converted into notes.

When adding the functionality for Anki notes, the database provided by the notes themselves was a great starting point. The csv files rely on the notes files for their creation as it is important to include any changes that the user may have made after the notes were generated. I didn't want to create a mehtod for making the csv files, but there was a need and I also didn't feel that it'd be fair to non-Obsidian user.

Knowledge

As a Japanese learner, I myself don't know much Japanes, but I do know that there are many smart people who have already done a lot to provide learners with information and have done so for free. Jisho says this about themselves on their website:

Jisho is a powerful Japanese-English dictionary. It lets you find words, kanji, example sentences and more quickly and easily. Our goal is to build a new kind of dictionary that doesn't just let you look up single words or kanji, but rather helps you understand any Japanese text. Ultimately we want to see it be a Google Now or Wolfram Alpha for the Japanese language. Just paste what you want to understand into Jisho, be it English, romaji, a single word or an entire paragraph of Japanese text, and it will search a myriad of data to help you understand the words, kanji and even grammar patterns

I personally see Jisho as the single best thing that has happened for Japanese language learners that's method agnostic. It doesn't matter what learners believe. If you need a general knowledge base of Japanese, Jisho will always be at the top of recommendations for resources, so I thank the contributors to Jisho with all my heart.

Google Translate was a forced option. I wanted something that was free and allow the project to be open source. For some reason, that last part is a bit hard as most translation services require an API key no matter the usage. I don't want a service that could have exposed someone's API key so Google Translate is used for sentence translation.

With data from these sources, the learner is given a good starting point.
